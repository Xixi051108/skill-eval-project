# 数据源接入规范

> 本 Skill 统一数据接入出口：本地文件或粘贴 JSON 最终都转成同一份 `extracted_data.json`（StandardRecords）供分析/渲染使用。学生环境不支持线上数据库或内部 HTTP API 查询。

---

## 1. 标准 StandardRecords 结构

```jsonc
{
  "meta": {
    "products": ["TDSQL-B v22.7.2", "TDSQL-B v22.7.3"],
    "scenarios": ["oltp_point_select", "oltp_read_only", "oltp_write_only", "oltp_read_write", "oltp_update_index"],
    "concurrencies": [32, 64, 128, 256, 512, 1000],
    "test_env": {},
    "log_meta": [...],
    "source_info": {
      "type": "local_file | local_data",
      "value": "xxx",
      "rows_fetched": 123
    }
  },
  "records": [
    {
      "product": "...",
      "scenario": "oltp_point_select",
      "threads": 128,
      "tps": 12345,
      "qps": 12345,
      "p95_ms": 1.23,
      "p99_ms": 2.34,
      "start_time": "..."
    }
  ]
}
```

允许 `records` 中保留来自原始文件的额外字段（例如 `raw`、`source_log`），但报告正文只能引用已核验的数值字段。

## 2. 数据源类型派发

`scripts/data_source_adapter.py` 根据 `intent.data_source_type` 派发：

| data_source_type | 适配器函数 | 说明 |
|------------------|-----------|------|
| `local_file` (.log) | `adapter_local_log` | 复用 `tdsql-b-whitepaper/scripts/log_to_excel.parse_log` |
| `local_file` (.xlsx) | `adapter_local_xlsx` | 按既定 Sheet/表头读取 |
| `local_file` (.json) | `adapter_local_json` | 标准 records 或本地导出的原始测试行数组 |
| `local_file` (.csv) | `adapter_local_csv` | 表头与 records 字段一致 |
| `local_data` | `adapter_raw_rows` 或直接返回标准 records | 粘贴 JSON，可为标准 records 或原始测试行 |
| `missing_data` | 无 | 停止并要求用户提供本地数据 |

## 3. 本地文件格式

### 3.1 `.log`

适用于 sysbench OLTP 日志，至少包含：

```text
Sysbench oltp Test
Time: 2026-06-11 09:30:00
DB: TDSQL-B-v22.7.3 host=10.0.0.12:3306
Data: tables=16, table_size=1000000, prepare=true
Threads: 32 64 128 256 512 1000
Duration: 300s
--- oltp_point_select | threads=32 | ... TPS=19850.20, QPS=19850.20, P95=1.95ms, P99=3.10ms
```

### 3.2 `.json`

优先推荐标准 records JSON：

```json
{
  "meta": {
    "products": ["TDSQL-B v22.7.3"],
    "scenarios": ["oltp_point_select"],
    "concurrencies": [32],
    "source_info": {"type": "local_file", "value": "mock.json", "rows_fetched": 1}
  },
  "records": [
    {"product": "TDSQL-B v22.7.3", "scenario": "oltp_point_select", "threads": 32, "tps": 19850.2, "qps": 19850.2, "p95_ms": 1.95, "p99_ms": 3.1}
  ]
}
```

也可接收本地导出的原始测试行数组，适配器会读取其中的 `dimension`、`results`、`results_key` 字段并转成标准 records。

### 3.3 `.xlsx`

每个产品一个 Sheet，表头必须为：

```text
scenario | threads | tps | qps | p95_ms | p99_ms
```

### 3.4 `.csv`

必须包含：

```text
product,scenario,threads,tps,qps,p95_ms,p99_ms
```

## 4. 数据质量门控

`data_source_adapter.quality_check()` 要求：

- `records` 长度 > 0（否则 E2004）。
- `tps/qps/p95_ms` 空值率为 0%（否则 E2005）。
- 输出 `concurrencies_by_product_scenario` 供后续门控②展示。
- 对于 TPC-DS duration 数据，要求 `duration_s` 空值率为 0%。

---

## 5. 错误码

| 代码 | 含义 |
|------|------|
| E2001 | 本地文件不存在 |
| E2002 | 不识别的扩展名 |
| E2003 | 缺少本地数据源 |
| E2004 | 本地数据为空或无法转成 records |
| E2005 | 关键指标空值率超标 |

---

## 6. 禁止事项

- 禁止调用内部 HTTP API 查询数据。
- 禁止连接 PostgreSQL 或其他线上数据库补数据。
- 禁止根据 `task_id` / `plan_id` / `report_id` 推断报告数据。
- 禁止为了让报告通过而手写、估算或硬编码性能指标。
